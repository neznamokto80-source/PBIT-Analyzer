# Документация по структуре файлов PBIT

## Общая информация

Файлы `.pbit` представляют собой ZIP-архивы, содержащие несколько файлов без расширения, которые являются JSON в кодировке **UTF-16-LE** (иногда UTF-16). Основные файлы для анализа:

- **DataModelSchema** — содержит модель данных: таблицы, столбцы, меры, связи, выражения, настройки.
- **Report/Layout** — содержит layout отчета: секции, визуальные элементы, фильтры, настройки темы.

## DataModelSchema

### Ключи верхнего уровня

- `name` — идентификатор модели (GUID)
- `compatibilityLevel` — уровень совместимости Power BI (например, 1601)
- `model` — объект модели (основной контент)

### Структура `model`

- `culture` — локаль (например, "ru-RU")
- `dataAccessOptions` — настройки доступа к данным
- `defaultPowerBIDataSourceVersion` — версия источника данных
- `sourceQueryCulture` — локаль для запросов
- `tables[]` — массив таблиц
- `relationships[]` — массив связей между таблицами
- `cultures[]` — поддерживаемые локали
- `expressions[]` — глобальные выражения
- `annotations[]` — аннотации модели

### Структура `tables[]` (каждая таблица)

- `name` — имя таблицы
- `lineageTag` — тег происхождения
- `columns[]` — массив столбцов
- `partitions[]` — массив разделов
- `measures[]` — массив мер
- `annotations[]` — аннотации таблицы
- `isHidden` — скрыта ли таблица
- `description` — описание таблицы
- `sourceLineageTag` — тег происхождения источника

### Структура `columns[]`

- `name` — имя столбца
- `description` — описание столбца
- `dataType` — тип данных ("int64", "double", "string", "dateTime", "boolean", "currency", "binary")
- `type` — тип столбца ("calculated", "calculatedTableColumn", "rowNumber", "data")
- `expression` — DAX-выражение для вычисляемых столбцов
- `formatString` — строка форматирования (например, "0.00%")
- `sortByColumn` — имя столбца, по которому производится сортировка
- `summarizeBy` — агрегация по умолчанию ("sum", "average", "count", ...)
- `lineageTag` — тег происхождения
- `annotations[]` — аннотации столбца

### Структура `measures[]`

- `name` — имя меры
- `expression` — DAX-выражение
- `formatString` — строка форматирования
- `formatStringDefinition` — определение формата (объект)
- `description` — описание меры
- `lineageTag` — тег происхождения
- `annotations[]` — аннотации меры

### Структура `partitions[]`

- `name` — имя раздела
- `source` — объект источника
  - `type` — тип источника ("m", "calculated", "entity", "structuredData")
  - `expression` — выражение источника (M или DAX)
- `mode` — режим раздела

### Структура `relationships[]`

- `name` — имя связи
- `fromTable` — исходная таблица
- `fromColumn` — исходный столбец
- `toTable` — целевая таблица
- `toColumn` — целевой столбец
- `isActive` — активна ли связь
- `crossFilteringBehavior` — поведение перекрестной фильтрации

### Структура `expressions[]`

- `name` — имя выражения
- `expression` — DAX-выражение
- `description` — описание

## Layout (Report/Layout)

### Ключи верхнего уровня

- `id` — идентификатор отчета (обычно 0)
- `filters` — фильтры уровня отчета (JSON строка)
- `resourcePackages[]` — массив ресурсных пакетов
- `sections[]` — массив секций (страниц)
- `config` — конфигурация отчета (JSON строка)
- `layoutOptimization` — уровень оптимизации layout

### Структура `resourcePackages[]`

- `resourcePackage` — объект пакета
  - `name` — имя пакета
  - `type` — тип (1 — зарегистрированные ресурсы, 2 — общие ресурсы)
  - `items[]` — массив элементов (изображения, темы)
  - `disabled` — отключен ли пакет

### Структура `sections[]` (каждая секция)

- `id` — идентификатор секции
- `name` — внутреннее имя
- `displayName` — отображаемое имя (название страницы)
- `filters` — фильтры секции (JSON строка)
- `ordinal` — порядковый номер
- `visualContainers[]` — массив визуальных контейнеров
- `config` — конфигурация секции (JSON строка)
- `displayOption` — опция отображения
- `width`, `height` — размеры секции

### Структура `visualContainers[]`

- `x`, `y`, `z` — координаты
- `width`, `height` — размеры
- `config` — конфигурация визуала (JSON строка)
- `filters` — фильтры визуала (JSON строка)
- `tabOrder` — порядок табуляции

#### Конфигурация визуала (`config`)

После парсинга JSON содержит:

- `name` — имя визуала
- `layouts[]` — массив layout'ов
- `singleVisual` — объект визуала
  - `visualType` — тип визуала ("slicer", "barChart", "lineChart", "table", ...)
  - `projections` — проекции (роли данных)
  - `prototypeQuery` — прототип запроса
  - `expansionStates` — состояния развертывания
  - `drillFilterOtherVisuals` — влияет ли детализация на другие визуалы
  - `objects` — объекты стилей и форматов
  - `filterExpressionMetadata` — метаданные фильтров

## Текущее извлечение в PBIT Analyzer

### DataModelSchema (через DataModelExtractor)

#### Таблицы (справочник таблиц)
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `name` | Да | Выводится как «Имя таблицы» |
| `isHidden` | Частично | Используется для фильтрации (скрытые таблицы не включаются в отчёт) |
| `lineageTag` | Нет | Не выводится |
| `description` | Да | Выводится как «Описание» |

#### Столбцы
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `name` | Да | Выводится как «Столбец» |
| `description` | Да | Выводится как «Описание» |
| `dataType` | Да | Выводится как «Тип данных» |
| `type` | Да | Используется для определения «Тип значения» |
| `expression` | Да | Выводится как «Формула (если вычисляемый)» |
| `formatString` | Да | Выводится как «Формат» |
| `sortByColumn` | Да | Выводится как «Сортировка по столбцу» |
| `summarizeBy` | Да | Выводится как «Агрегация по умолчанию» |
| `lineageTag` | Нет | Не извлекается |
| `annotations` | Нет | Не извлекается |

#### Меры
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `name` | Да | Выводится как «Мера» |
| `description` | Да | Выводится как «Описание» |
| `expression` | Да | Выводится как «Формула (DAX)» |
| `formatString` | Да | Извлекается из `formatStringDefinition` и `annotations`, выводится как «Формат» |
| `displayFolder` | Да | Выводится как «Папка отображения» |
| `lineageTag` | Нет | Не извлекается |
| `annotations` | Нет | Не извлекается (кроме аннотации «Format») |

#### Источники
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `type` | Да | Выводится как «Тип источника» |
| `expression` | Да | Форматируется и выводится как «Как создается» |
| `partitions` детально | Нет | Извлекается только тип источника |

#### Связи
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `fromTable` | Да | Входит в состав поля «Откуда» |
| `fromColumn` | Да | Входит в состав поля «Откуда» |
| `toTable` | Да | Входит в состав поля «Куда» |
| `toColumn` | Да | Входит в состав поля «Куда» |
| `isActive` | Да | Выводится как «Состояние» |
| `crossFilteringBehavior` | Нет | Не извлекается |

#### Глобальные выражения и другие поля
- `expressions` (глобальные) — не извлекаются
- `dataAccessOptions`, `sourceQueryCulture`, `cultures` — не извлекаются

### Layout (через LayoutExtractor)

#### Секции (листы)
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `displayName` | Да | Выводится как «Имя листа» |
| `ordinal` | Да | Выводится как порядковый номер |
| `name` (внутреннее) | Да | Выводится как «ID листа» |
| `visibility` | Да | Выводится как «Видимость» |
| `width`, `height` | Да | Выводятся как объединённый столбец «Размеры» (формат «ширина x высота») |
| `config` | Нет | Не извлекается (сырая строка) |

#### Визуальные элементы
| Поле | Извлекается | Примечание |
|------|-------------|------------|
| `visualType` | Да | Выводится как «Тип визуала» |
| `projections` | Да | Используется для извлечения ролей и полей |
| `prototypeQuery` | Да | Используется для извлечения мер |
| `objects` | Да | Используется для условного форматирования |
| `FilterTargetEntity` | Да | Выводится как отдельный столбец |
| `FilterTargetProperty` | Да | Выводится как отдельный столбец |
| `FilterExpressionShort` | Да | Выводится как отдельный столбец |
| `filters` (сырая строка) | Нет | Не парсится, но используется для извлечения FilterTargetEntity и т.д. |
| `config` → `themeCollection` | Нет | Не извлекается |
| `displayOption` | Нет | Не извлекается |
| `vcObjects` (детальные стили) | Нет | Не извлекаются |

#### Условное форматирование
- Извлекается из `objects` визуалов, выводится в отдельном листе «Условное форматирование».

#### Не извлекаются (общее)
- Фильтры уровня отчета, секции (только как сырая строка)
- Ресурсные пакеты
- Конфигурация темы (`config` → `themeCollection`)
- Параметры отображения (`displayOption`, `width`, `height`) секций
- Детальные объекты стилей (`vcObjects`)

## Поля, рекомендуемые для добавления

### DataModelSchema

1. **lineageTag** — для отслеживания происхождения (таблицы, столбцы, меры)
2. **annotations** — дополнительные метаданные (можно извлекать как JSON)
3. **expressions** — глобальные выражения (переменные)
4. **partitions детально** — тип, выражение, параметры
5. **isHidden** — отдельная категория для скрытых объектов (вывод в отчёт)
6. **crossFilteringBehavior** — поведение перекрестной фильтрации связей

### Layout

1. **filters** — парсить JSON строку и извлекать выражения фильтров, типы, параметры (уровень отчета, секции)
2. **resourcePackages** — список ресурсов (изображения, темы) с путями
3. **config** — настройки темы (`themeCollection`), параметры отчета
4. **displayOption**, **width**, **height** секций — для понимания layout'а
5. **visualContainers.config** → **objects** — больше деталей о стилях, границах, фоне
6. **visualContainers.config** → **filterExpressionMetadata** — метаданные фильтров визуала

## Примеры путей к данным

### DataModelSchema

- `model.tables[0].name` — имя первой таблицы
- `model.tables[0].columns[0].formatString` — формат первого столбца
- `model.tables[0].measures[0].description` — описание первой меры
- `model.relationships[0].crossFilteringBehavior` — поведение фильтрации связи
- `model.expressions[0].expression` — глобальное выражение

### Layout

- `sections[0].displayName` — название первой страницы
- `sections[0].visualContainers[0].config` → `singleVisual.visualType` — тип первого визуала
- `sections[0].visualContainers[0].filters` — фильтры первого визуала (JSON строка)
- `config` → `themeCollection.baseTheme.name` — имя базовой темы

## Заключение

Данная документация может использоваться для дальнейшего расширения функциональности PBIT Analyzer. При добавлении новых полей следует учитывать, что структура файлов может незначительно отличаться в зависимости от версии Power BI, поэтому необходима обработка отсутствующих ключей.

Рекомендуется создать единый модуль для парсинга этих структур с возможностью извлечения всех перечисленных полей по требованию (например, через флаги извлечения).
