# Предложения по улучшению проекта PBIT Analyzer

## Обзор проекта

PBIT Analyzer — это desktop-приложение на Python с GUI на PyQt6 для анализа файлов шаблонов Power BI (.pbit). Приложение распаковывает архив, читает файлы DataModelSchema и Layout в кодировке UTF-16-LE, извлекает метаданные (таблицы, столбцы, меры, связи, источники, визуальные элементы, условное форматирование) и предоставляет их в виде таблиц с возможностью построения графа зависимостей (lineage) и генерации Excel-отчета.

## Выявленные возможности для расширения

На основе анализа структуры файлов DataModelSchema и Layout обнаружены дополнительные поля и структуры, которые в настоящее время не извлекаются или не отображаются.

### 1. Дополнительные данные из DataModelSchema

- **lineageTag** и **sourceLineageTag** у таблиц, столбцов, мер — полезны для отслеживания происхождения данных.
- **annotations** — содержат метаданные, описания, теги.
- **description** — текстовое описание таблиц, столбцов, мер.
- **formatString** — строки форматирования для столбцов и мер (числа, даты, валюта).
- **sortByColumn** — указание столбца для сортировки.
- **summarizeBy** — агрегация по умолчанию.
- **dataAccessOptions**, **sourceQueryCulture**, **cultures** — настройки доступа к данным и локализации.
- **expressions** — глобальные выражения (переменные, вычисляемые объекты).
- **partitions** — детальная информация о разделах таблиц (тип, запрос, параметры).
- **isHidden** — флаг скрытых таблиц/столбцов (сейчас они фильтруются, но можно добавить отдельную вкладку).

### 2. Дополнительные данные из Layout

- **filters** на уровне отчета, секции, визуала — выражения фильтров, типы, параметры.
- **config** — настройки темы (themeCollection), параметры отображения, настройки фильтрации.
- **resourcePackages** — ресурсы (изображения, темы), используемые в отчете.
- **displayOption**, **width**, **height** — размеры и параметры секций.
- **visualContainers** — более детальная информация о визуалах (проекции, стили, объекты, условное форматирование). Частично извлекается, но можно добавить больше полей.

## Предложения по улучшению извлечения данных

### Новые вкладки/таблицы

1. **Метаданные** — таблица с аннотациями, описаниями, lineageTag, formatString для всех объектов.
2. **Фильтры** — иерархия фильтров (отчет → секция → визуал) с выражениями и типами.
3. **Ресурсы** — список resourcePackages (изображения, темы) с путями и типами.
4. **Разделы таблиц (Partitions)** — детализация partitions: тип источника, выражение, параметры.
5. **Скрытые объекты** — отдельная вкладка для таблиц и столбцов с isHidden = true.

### Улучшение существующих вкладок

- **Столбцы** — добавить колонки: formatString, sortByColumn, summarizeBy, lineageTag.
- **Меры** — добавить колонки: formatString, description, lineageTag.
- **Таблицы** — добавить колонки: description, lineageTag, sourceLineageTag.
- **Визуальные элементы** — добавить колонки: фильтры визуала, настройки стилей, проекции (queryRef).

## Предложения по улучшению пользовательского интерфейса

1. **Добавление вкладки "Метаданные"** — объединить все дополнительные метаданные в одной таблице с фильтрацией по типу объекта.
2. **Вкладка "Фильтры"** — древовидное представление фильтров с возможностью раскрытия выражений.
3. **Вкладка "Ресурсы"** — список ресурсов с предпросмотром изображений (если возможно).
4. **Улучшение lineage** — визуализация графа зависимостей с помощью библиотеки graphviz или интерактивного виджета (например, pyvis). Возможность экспорта графа в PNG/SVG.
5. **Поиск и фильтрация** — добавить строку поиска в каждую вкладку для быстрого нахождения элементов по имени, ID, типу.
6. **Экспорт данных** — поддержка дополнительных форматов экспорта: CSV, JSON, HTML-отчет.
7. **Настройки пользователя** — окно настроек для выбора отображаемых колонок, темы интерфейса, сохранения последнего каталога.
8. **Подсветка синтаксиса DAX** — в ячейках с формулами использовать подсветку (например, через QSyntaxHighlighter).
9. **Контекстное меню** — в таблицах добавить правый клик с опциями: "Копировать ID", "Копировать имя", "Копировать формулу", "Найти lineage".
10. **Статусная строка** — отображать информацию о загруженном файле: количество таблиц, столбцов, мер, визуалов, время загрузки.

## Предложения по улучшению производительности и обработки ошибок

1. **Кэширование результатов анализа** — сохранять распакованные данные и промежуточные структуры в кэш (например, pickle) с хэшем файла. При повторном открытии того же файла загружать из кэша. - не делать
2. **Фоновая обработка** — вынести тяжелые операции (парсинг JSON, построение lineage) в отдельные потоки (QThread) или процессы, чтобы GUI оставался отзывчивым.
3. **Детальный прогресс-бар** — разбить процесс анализа на этапы с оценкой времени для каждого этапа.
4. **Обработка больших файлов** — использовать потоковое чтение JSON (ijson) для уменьшения потребления памяти. Оптимизировать структуры данных (использовать более эффективные контейнеры).
5. **Расширенное логирование** — добавить уровни логирования (DEBUG, INFO, WARNING, ERROR) с возможностью сохранения в файл. В GUI добавить окно просмотра логов. - не делать
6. **Улучшение определения кодировки** — в функции `load_json_utf16` добавить автоматическое определение BOM, fallback на бинарный анализ и более широкий список кодировок. -не делать
7. **Валидация структуры JSON** — проверять наличие обязательных полей, обрабатывать отсутствующие ключи gracefully, предоставлять понятные сообщения об ошибках.-не делать
8. **Многопоточное извлечение** — параллельное извлечение данных из таблиц и визуалов (где нет зависимостей) для ускорения на многоядерных системах.
9. **Оптимизация построения lineage** — использовать индексы для быстрого поиска зависимостей, алгоритмы обхода графа с кэшированием результатов.
10. **Надежная очистка временных файлов** — использовать контекстные менеджеры и `try...finally` для гарантированного удаления временных каталогов даже при возникновении исключений.

## Заключение

Реализация даже части предложенных улучшений значительно повысит полезность инструмента для аналитиков и разработчиков Power BI, позволит глубже понимать структуру отчетов и ускорит процесс анализа.

Предложения сгруппированы по приоритету: сначала рекомендуется реализовать извлечение дополнительных метаданных и улучшение UI, затем оптимизацию производительности.
